## 一、项目目标与应用场景

### 1.1 项目定位

ClawTeam 是一个基于 MoonBit 构建的**纯调度层工具**，核心价值在于**通过聊天式交互统一编排外部 CLI 工具**（Claude Code、Codex、Gemini、OpenClaw、OpenCode、Qwen 等）。

> 🎯 核心定位：仅作调度，不涉业务  
> - **ClawTeam 负责**：消息路由、任务拆解、PTY 调用、输出捕获、多渠道接入、定时触发、执行审计  
> - **外部 CLI 负责**：安全策略、工作空间管理、文件操作、模型调用、工具执行、状态维护等**全部业务逻辑**

### 1.2 核心应用场景：内容创作者的自动化工作流

本项目重点围绕**内容创作者**的日常高频需求，构建一套可交付的自动化工作流。

#### 场景描述
内容创作者需要每日发布技术博客，流程涉及：选题调研、大纲生成、初稿撰写、代码示例验证、SEO 优化、格式排版、发布推送。传统流程需人工切换多个工具，ClawTeam 将其编排为单一自动化任务。

#### 可交付流程详解

| 阶段 | 用户操作/编排 | 前端交互 | 后端流程 (ClawTeam) | 外部 CLI 执行 (业务逻辑) | 期望输出 |
|------|--------------|----------|---------------------|-------------------------|----------|
| **1. 任务定义** | 用户在 Web UI 配置"每日博客" Job，设置 Cron 表达式，绑定 CLI 链 | 可视化工作流编辑器，拖拽 CLI 节点，配置参数模板 | 解析 Job 定义，存储至 `~/.clawteam/jobs/`，验证 CLI 路径可用性 | 无 | 任务配置持久化，状态显示"已就绪" |
| **2. 触发执行** | 定时触发或手动点击"立即运行" | 状态变更为"运行中"，实时日志流窗口打开 | 读取 Job 定义，按顺序/并行拉起 PTY 进程，注入上下文参数 | 无 | 进程启动，PID 记录 |
| **3. 调研与大纲** | 无介入 (自动) | 显示 `Claude Code` 实时输出流 | 调用 `claude-code "调研 Rust 异步最新进展并生成大纲"` | 联网搜索、阅读文档、生成 Markdown 大纲 | 大纲文件 `outline.md` |
| **4. 初稿与代码** | 无介入 (自动) | 显示 `Codex` 实时输出流 | 调用 `codex "基于大纲撰写正文，包含可运行代码示例"` | 撰写内容、生成代码文件、本地运行验证代码 | 草稿文件 `draft.md` + 代码文件 |
| **5. 审核与优化** | **用户介入点**：审核草稿 | 显示 `Gemini` 输出，提供"通过/修改"按钮 | 调用 `gemini "审查草稿准确性与 SEO"`，暂停等待用户确认 | 审查内容、提出修改建议、生成 SEO 关键词 | 审查报告 `review.json` |
| **6. 发布与归档** | 无介入 (自动) | 显示 `Qwen` 输出，最终状态"成功" | 调用 `qwen "格式化 Markdown 并调用发布 API"` | 格式排版、调用 WordPress/GitHub API 发布 | 发布链接 + 归档日志 |

#### 追踪审计模块 (Audit & Tracking) 设计

针对上述流程，ClawTeam 提供独立的**执行审计模块**，不负责业务内容审计，仅负责**调度行为审计**。

| 审计维度 | 追踪内容 | 发生时机 | 自动化/人工 | 可配置性 | 用户介入 |
|----------|----------|----------|-------------|----------|----------|
| **调度审计** | Job 触发时间、触发源 (Cron/Manual)、调度延迟 | 任务触发瞬间 | 自动记录 | 可开启/关闭日志 | 仅查看 |
| **进程审计** | CLI 进程 PID、启动命令、环境变量哈希、退出码 | 进程生命周期内 | 自动捕获 | 敏感信息脱敏配置 | 无 |
| **资源审计** | 进程运行时长、CPU/内存峰值 (可选)、PTY 缓冲区大小 | 进程运行中 | 自动监控 | 监控频率配置 | 无 |
| **输出审计** | stdout/stderr 完整日志、关键词匹配 (如 "Error") | 实时流式捕获 | 自动存储 | 日志保留策略 (7 天/30 天) | 可搜索/下载 |
| **干预审计** | 用户暂停、恢复、终止操作的时间点与操作人 | 用户点击控制按钮时 | 自动记录 | 无 | **手动暂停/终止** |
| **异常审计** | CLI 崩溃、超时、非零退出、PTY 断开 | 异常发生时 | 自动告警 | 告警阈值配置 | 接收通知 |

- **用户介入时机**：仅在配置了"人工确认节点"的工作流中（如场景阶段 5），ClawTeam 会暂停后续 CLI 调用，等待用户通过前端点击"继续"或"终止"。
- **中断能力**：用户可随时通过前端"终止"按钮，ClawTeam 向对应 PTY 进程发送 `SIGINT` 信号，记录终止审计日志。
- **审计配置**：支持通过 `config.yaml` 配置审计日志级别（Info/Warn/Error）、存储路径及保留周期。

### 2.2 功能边界

**✅ ClawTeam 负责（纯调度层）**：
- 聊天消息 → 命令行参数的转换与路由
- 基于 PTY 的外部 CLI 调用与输出流捕获
- Job 定义解析、定时触发、执行状态记录（仅日志层面）
- 多渠道消息接入与事件转发
- 执行审计日志的记录与存储

**❌ 外部 CLI 负责（执行层，完全自治）**：
- 安全策略与权限控制
- 工作空间/文件系统的读写操作
- 模型调用、Prompt 工程、工具执行逻辑
- 错误处理、重试策略、资源配额管理
- 任何业务相关的状态维护与持久化

**❌ 不包含范围**：
- ❌ 原生 CLI 交互界面（用户不直接使用终端操作 ClawTeam）
- ❌ 工作空间状态同步、文件管理、检查点等能力
- ❌ 云端 CLI 托管与分布式执行
- ❌ 用户账号体系与 SSO 集成
- ❌ 商业化计费、配额管理与审计日志

### 3.2 关键技术选型与调研结论

#### 3.2.1 核心语言与异步运行时
- **选型**：MoonBit (Native Target) + `moonbitlang/async`
- **调研结论**：
    - MoonBit 原生目标可直接调用系统 API（如 `fork`, `exec`, `pty`），无需 FFI 桥接，性能优于 WASM 方案。
    - `moonbitlang/async` 提供轻量级协程模型，适合高并发 CLI 调度场景。相比传统线程，协程上下文切换开销更低，适合维持数十个并发 PTY 会话。
    - **异步逻辑处理**：采用 `async/await` 语法糖，每个 CLI 会话作为一个独立 Task 运行，通过 `Channel` 或 `Stream` 进行输出流通信，避免回调地狱。

#### 3.2.2 前端与后端通信
- **选型**：HTTP REST API (控制) + SSE (Server-Sent Events) / WebSocket (日志流)
- **调研结论**：
    - **控制指令**（启动/停止/配置）：使用 HTTP POST/GET，实现简单，兼容性好。
    - **日志流**（CLI 输出）：使用 SSE 或 WebSocket。鉴于 MoonBit `internal/httpx` 已支持 `EventStream`，优先选用 SSE，减少前端连接维护成本，单向推送足够满足日志需求。
    - **FFI 边界**：若前端采用 Web 技术（React/Vue），不存在 FFI，纯 HTTP 通信。若采用 Tauri (Rust + Web)，MoonBit 后端作为独立进程运行，通过 Loopback HTTP 通信，避免复杂的 WASM/JS FFI 绑定，确保 PTY 权限不受沙箱限制。

#### 3.2.3 组件模型
- **选型**：Trait -based Component System (MoonBit Interface)
- **调研结论**：
    - MoonBit 的 `trait` 机制适合定义 `Channel` 和 `CLI Adapter` 接口。
    - **Channel 组件**：定义 `send`, `receive`, `handle_event` 接口，Feishu/Weixin 实现该接口，实现插件化扩展。
    - **CLI 组件**：定义 `execute`, `parse_output` 接口，不同 CLI 工具实现该接口，便于统一调度。

#### 3.2.4 后端调用 CLI 工具
- **选型**：MoonBit Native System Calls + `internal/fsx` (extended)
- **实现方式**：
    - 利用 MoonBit 原生目标能力，直接调用 libc `forkpty` 或等价系统调用。
    - 封装 `PTYSession` 结构体，管理文件描述符 (FD)，实现非阻塞读写。
    - **调研结论**：MoonBit 标准库尚在完善中，需基于 `internal/fsx` 扩展 PTY 功能。参考 MoonClaw 的 `agent` 子系统工具调用逻辑，但改为外部进程调用而非内部函数调用。

### 3.3 大模型与智能体工具的作用

| 开发阶段 | 大模型/智能体应用 | 价值 |
|----------|------------------|------|
| **命令行组装** | LLM 辅助将用户聊天转换为 CLI 参数模板 | 降低配置门槛，提升易用性 |
| **输出解析** | LLM 辅助结构化不同 CLI 的文本输出 | 提升反馈可读性（可选增强） |
| **异常诊断** | 基于执行日志的失败原因分析建议 | 辅助用户快速定位问题（不自动修复） |
| **文档生成** | 自动从执行日志生成操作摘要 | 降低维护成本 |

## 四、风险分析与应对方案

### 4.1 技术风险

| 风险项 | 影响 | 缓解措施 |
|--------|------|----------|
| **MoonBit 原生 PTY 支持不足** | 无法稳定捕获 CLI 输出或控制进程 | 基于 `internal/fsx` 自行封装 PTY 模块 + 参考 MoonClaw 底层实现 + 预留 Rust FFI 降级方案 |
| **外部 CLI 输出格式不稳定** | 解析失败导致反馈文本混乱 | 输出解析层 + 降级策略（原始文本回传）+ 可选 LLM 辅助 |
| **外部 CLI 行为不可控** | 安全、文件操作等由 CLI 决定，ClawTeam 无法干预 | 明确责任边界 + 执行前用户确认（可配置）+ 操作日志记录 |
| **多 CLI 并发资源竞争** | 文件锁冲突、端口占用（由 CLI 自身管理） | 提供资源配额建议 + 执行队列调度 + 冲突检测提示 |

### 4.2 工程风险

| 风险项 | 影响 | 缓解措施 |
|--------|------|----------|
| **MoonBit 生态成熟度** | 第三方库不足，开发效率受影响 | 核心依赖自研（internal/）+ 社区协作 + 渐进式迁移策略 |
| **多渠道接入复杂度** | 飞书/微信协议差异大，维护成本高 | 抽象 Channel Trait + 插件化实现 + 核心渠道优先 |
| **外部 CLI 依赖风险** | 某 CLI 停止维护或变更协议 | 支持多备选 CLI + 适配器配置抽象 + 社区维护机制 |

### 4.3 替代方案预案

- 若 PTY 方案在 Windows 平台遇到不可解兼容问题：降级为 `subprocess` + 流重定向，牺牲部分交互能力保核心功能
- 若外部 CLI 输出解析效果未达预期：提供"原始输出模式"，直接回传 CLI 原始文本
- 若多 CLI 协同效果未达预期：提供"单 CLI 模式"，确保基础可用性

### 5.2 技术现状理解

- **纯调度层设计**：当前主流方案（如 LangChain、AutoGen）侧重 Python 生态，缺乏系统级资源管控。ClawTeam 基于 MoonBit 原生编译，提供更强的执行隔离与调度效率。
- **聊天驱动编排**：现有工具多依赖 YAML/JSON 静态配置，缺乏"聊天即编排"的交互范式。本项目通过消息→命令行模板转换，降低用户使用门槛。
- **渠道集成**：企业微信/飞书等 IM 平台已开放丰富 API，但缺乏统一的 CLI 接入标准。ClawTeam 的 Channel Trait 设计可为社区提供扩展规范参考。

### 5.3 差异化创新点

1. **极简调度层**：仅负责命令行调用与输出捕获，职责边界清晰，易于维护与扩展
2. **无数量限制的 CLI 并行**：基于轻量协程 + PTY 池化，突破传统终端的单实例限制
3. **多渠道统一入口**：飞书/微信/Web 消息统一路由至 CLI，实现"一处配置，多处使用"
4. **零迁移成本**：完全兼容现有 CLI 工具，用户无需学习新语法即可享受协同能力
5. **完全自治的执行层**：安全、文件、模型等能力由 CLI 自身实现，ClawTeam 不侵入业务逻辑

### 6.2 核心代码结构示意

```
clawteam/
├── scheduler/        # 核心：消息→命令行组装 + 路由
├── pty/              # 复用 MoonClaw PTY 引擎（精简版）
├── chat/             # 聊天中枢：消息收发 + 上下文透传
├── job/              # 复用 MoonClaw Job 触发器（仅定时执行）
├── channel/          # 复用渠道抽象，适配新交互模式
├── ui/               # 前端：多 CLI 聊天界面
├── parser/           # 输出解析：正则 + 可选 LLM 辅助
├── logger/           # 日志持久化：仅记录，不干预
├── cmd/
│   └── clawteam.mbt  # 主入口：gateway + chat 模式
└── internal/         # 工具库：httpx, fsx, pino 等
```
